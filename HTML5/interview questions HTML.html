- HTML stands for HyperText Markup Language, used to create web pages.

- HTML comprises tags, defining page structure and content.

- The DOCTYPE specifies HTML version, aiding correct rendering.

- Elements form page components, tagged for structure, attributes provide extra info.

- Common tags include headings, paragraphs, links, images, lists, tables.

- The <head> tag contains document metadata like title, charset.

- The <body> tag holds main visible content for display.

- Block-level elements start new lines, occupy full width. Inline don't.

- <div> groups elements for styling, layout structuring.

- <span> applies styles to specific text portions.

- <a> creates hyperlinks to other pages or files

- href specifies hyperlink destination URL.

- <img> displays images on web pages.

- src specifies image file or URL source.

- <table> creates tabular data structures.

- <thead> groups table header, <tbody> body, <tfoot> footer

- <tr> signifies a table row.

- <th> defines table header cell, <td> table data cell.

- colspan spans cells horizontally, rowspan vertically.

- <form> creates interactive input forms for users.

- Input, checkboxes, radio buttons, dropdown lists, text areas.

- name identifies form elements for server-side processing.

- method determines how form data is sent (GET or POST).

- action specifies where form data is sent upon submission.

- <input> creates various form input fields.

- type determines input field type (text, checkbox, radio, etc.).

- <label> associates text with form elements for accessibility.

- <select> creates dropdown selection lists.

- &lroption> defines selectable options within a dropdown.

- value assigns a value to an option, sent upon form submission.

- <textarea> creates multiline text input fields.

- <iframe> embeds external content within a page.

- Semantic HTML uses elements with meaningful roles

- <meta> provides document metadata like charset or viewport.

- The viewport tag controls mobile viewport width and scaling.

- alt provides alternative text for images, aiding accessibility

- title provides additional information or tooltips for elements.

- <fieldset> groups related form elements, <legend> provides caption.

- <datalist> provides predefined input options for <input> fields.

- <meter> visualizes scalar measurements like progress bars.

- <time> represents specific dates or times for content.

- The required attribute mandates filling form fields before submission.

- autocomplete controls form field autocomplete behavior.

- <nav> defines navigation links section in a webpage.

- <abbr> defines abbreviations with optional explanations.

- <pre> displays preformatted text preserving whitespace

- disabled prevents interaction with form fields or buttons.

- readonly makes form fields non-editable but visible.

- <progress> visually represents task progress or completion.

- placeholder offers hints or examples in input fields.

- <ruby> annotates East Asian characters with pronunciation guides.

- <details> creates collapsible sections with <summary> heading.

- <wbr> suggests word break opportunities within long strings.

- contenteditable makes elements editable directly in the browser.

- spellcheck enables/disables spell checking in form fields.

- <cite> marks titles or creative work references.

- download prompts download of linked file upon click

- <script> embeds or references JavaScript code.

- Inline JS is directly in HTML, external JS is in separate files.

- <noscript> provides content for non-JavaScript enabled browsers.

- defer delays script execution until HTML parsing completes.

- async allows script execution without blocking HTML parsing.

- <iframe> embeds external content within a webpage.

- sandbox restricts iframe content for security.

- autocomplete controls form field autocomplete behavior.

- <figure> encapsulates content with an optional caption.

- <meter& visually represents measurements within a range

- <time> specifies dates, times, or durations in content.

- required mandates completion of form fields before submission.

- autocomplete controls form field autocomplete behavior
